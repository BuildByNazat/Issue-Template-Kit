# Contributing Guidelines

First off, thank you for considering contributing to this repository! It's people like you that make open source such a great community.

## Basic Behavior Expectations
We want this repository to be a welcoming environment for everyone.
- Be respectful and considerate of others.
- Keep feedback constructive.
- Ask questions if you are unsure about anything! There is no such thing as a bad question.

## How to Report Issues
If you find a bug or something that isn't working right:
1. Check the existing issues to ensure it hasn't already been reported.
2. If it hasn't been reported, open a new issue.
3. Select the `Bug report` template.
4. Fill out all the requested information, including steps to reproduce the bug and your environment details.

## How to Suggest Improvements
If you have an idea for a new feature or an improvement to an existing one:
1. Check if a similar feature request already exists in the issues.
2. Open a new issue and select the `Feature request` template.
3. Clearly describe the problem your feature solves and propose a potential solution.

## How to Submit Pull Requests
To contribute code or documentation changes:
1. Fork the repository to your own GitHub account.
2. Clone your fork locally and create a new branch for your changes (`git checkout -b your-feature-branch`).
3. Make your changes and ensure they are clean and well-structured.
4. Commit your changes with clear, descriptive messages.
5. Push your branch to your fork (`git push origin your-feature-branch`).
6. Open a Pull Request against the `main` branch of this repository.
7. Fill out the Pull Request Template completely.
8. Wait for a maintainer to review your code. We may ask for some adjustments before merging!

Thank you for your contributions!
